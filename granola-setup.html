Granola Integration
Setup Guide

Connect Granola to automatically pull meeting notes into deal records — no copy-pasting, no manual logging.

1

Check your Granola plan

The Granola API requires a Business plan or higher. If you're already on Business or Enterprise, skip to Step 2.

To upgrade: open Granola → Settings → Billing → upgrade to Business.

2

Generate your Granola API key

  1. Open the Granola app on your Mac
  2. Go to Settings → Connectors → API keys
  3. Click Create new key
  4. Under access scopes, select Personal notes
  5. Click Generate API Key
  6. Copy the key — it begins with grn_
Important: You won't be able to see this key again after closing the dialog. Copy it to a safe place before proceeding.

Mike and Pete each need their own key. Pete's key only sees Pete's Granola notes; Mike's only sees Mike's.
3

Add the key to the CRM

  1. Open the Part Human CRM
  2. Go to Settings (bottom of the left nav)
  3. Scroll to the Granola Integration section at the bottom
  4. Paste your key into the API Key field
  5. Click 🔌 Test connection — you should see a green confirmation with the number of notes accessible
  6. Click 💾 Save key
Confirmation: After saving, a green dot and message will appear: "API key saved. Open any deal → Meetings tab to sync Granola notes for that company."
4

Sync meetings on a deal

  1. Open any Deal from the pipeline
  2. Click the Meetings tab
  3. Click ↻ Granola in the top right of the Meeting Log
  4. The CRM scans your recent Granola notes, matches any involving this company, and imports them automatically
  5. A green confirmation shows how many meetings were imported
Syncing is always safe to repeat — already-imported notes are tracked and never duplicated. "Already up to date" just means nothing new was found.
?

How meetings get matched to deals

The CRM automatically identifies which Granola notes belong to which deal using these signals, in order of confidence:

Signal What it checks Confidence
Contact email Attendee email exactly matches the deal's contact email Definitive
Email domain Attendee email domain matches the company name (e.g. @stoneedge.com → Stone Edge) Strong
Contact name Attendee name matches the deal's contact name Moderate
Meeting title Meeting title contains the company name Supporting
Best practice: Make sure deals have a contact email filled in. Email matching is the most reliable signal and will catch the meeting even if the title doesn't mention the company.
!

Troubleshooting

"Invalid Granola API key"
The key was copied incorrectly or has been revoked. Generate a new one in Granola → Settings → Connectors → API keys.
"No new meetings found" but meetings are expected
Check that the deal has a contact email filled in. If the contact uses a personal email (Gmail, etc.) rather than a company address, also try adding their name to the deal's contact name field.
CORS or network error
The Granola API may not allow direct browser requests in certain environments. Contact Mike — a one-minute Supabase proxy fix has already been prepared in the code and just needs to be switched on.
↻ Granola button not visible
The button only appears after an API key has been saved in Settings. Confirm the Granola Integration section shows a green dot.
Meeting imported but no transcript
Granola only makes notes available via the API once an AI summary has been generated. Give Granola a minute to process after the meeting ends, then sync again.
A few things to know:
— Granola notes must have a completed AI summary to be available via the API.
— The sync scans up to 90 of your most recent notes — no need to sync immediately after every meeting.
— Meeting summaries, dates, and transcripts carry over. Action items do not — use the drag-and-drop transcript import on the Meetings tab for that.
— To remove the integration, go to Settings → Granola Integration → Remove key.
